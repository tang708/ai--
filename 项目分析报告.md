# AI健康管理系统 - 项目详细分析报告

---

## 一、项目概述

本项目是一个基于 **Spring Boot + Vue.js** 的前后端分离健康管理系统，集成了 **Spring AI (OpenAI)** 实现智能健康助手功能。系统支持用户健康数据管理、运动知识查询、AI健康建议等核心功能，并采用 **RBAC权限模型** 区分管理员与普通用户。

## 五、管理员与普通用户的区分机制（核心重点）

### 5.1 RBAC权限模型

本系统采用经典的 **RBAC（Role-Based Access Control，基于角色的访问控制）** 模型，通过5张表实现完整的权限体系：

```
用户(j_user) ──多对多──> 角色(j_role) ──多对多──> 菜单权限(j_menu)
         j_user_role              j_role_menu
```

**关键点：用户表（j_user）本身没有角色字段，角色信息通过中间表 j_user_role 关联。**

### 5.2 角色区分的具体实现

#### 数据库层面

| 角色 | role_id | role_name | role_desc | 可访问菜单 |
|------|---------|-----------|-----------|------------|
| 管理员 | 1 | admin | 超级管理员 | 系统管理(用户管理、角色管理、运动知识管理、运动详情管理、用户身体信息管理) + 系统功能(身体信息上传、健康评估、运动知识、身体信息管理) |
| 普通用户 | 3 | normal | 普通用户 | 系统功能(身体信息上传、健康评估、运动知识、身体信息管理) |

**管理员独有权限：** 用户管理、角色管理、运动知识管理、运动详情管理、用户身体信息管理

#### 后端层面

1. **登录时**：`UserServiceImpl.login()` 验证用户名密码后生成JWT Token，Token中包含用户基本信息（id、username等）

2. **获取用户信息时**：`UserServiceImpl.getUserInfo()` 解析Token后执行两步关键查询：
   - 调用 `UserMapper.getRoleNameByUserId(userId)` → 通过 `j_user_role` + `j_role` 联表查询获取角色名称列表
   - 调用 `MenuServiceImpl.getMenuListByUserId(userId)` → 通过 `j_user_role` + `j_role_menu` + `j_menu` 三表联查获取可访问菜单

3. **SQL查询核心**（MenuMapper.xml）：
```sql
SELECT a.*
FROM j_menu a, j_role_menu b, j_user_role c
WHERE a.menu_id = b.menu_id
  AND b.role_id = c.role_id
  AND c.user_id = #{userId}
  AND a.parent_id = #{pid}
```

4. **角色名查询SQL**（UserMapper.xml）：
```sql
SELECT b.role_name
FROM j_user_role a, j_role b
WHERE a.role_id = b.role_id
  AND a.user_id = #{userId}
```

#### 前端层面

1. **登录流程**：
   - 用户提交用户名密码 → 后端返回JWT Token + username + id
   - 前端将Token存入Cookie（key: `vue_admin_template_token`）

2. **权限路由生成**：
   - 前端路由守卫（`permission.js`）拦截每次路由跳转
   - 调用 `store.dispatch('user/getInfo')` 获取用户信息
   - 后端返回 `roles`（角色列表）和 `menuList`（菜单列表）
   - 前端根据 `menuList` 动态生成路由（`myFilterAsyncRoutes`函数）
   - 通过 `router.addRoutes()` 动态添加路由

3. **动态路由转换逻辑**：
   - `component === 'Layout'` → 映射为Layout组件
   - 其他 → 通过 `require('@/views/${menu.component}.vue')` 动态加载对应页面组件

4. **菜单渲染**：
   - 侧边栏根据动态路由自动渲染，管理员看到所有菜单，普通用户只看到系统功能菜单

### 5.3 注册时的角色分配

用户注册时（`UserServiceImpl.register()`），**默认分配 role_id=3（普通用户角色）**：

```java
UserRole userRole = new UserRole();
userRole.setUserId(userId);
userRole.setRoleId(3);  // 固定分配普通用户角色
userRoleService.save(userRole);
```

管理员角色只能通过后台管理界面手动分配。

### 5.4 权限控制总结图


## 六、前端网络连接机制详解

### 6.1 开发环境网络架构

```
浏览器 ──HTTP──> webpack-dev-server(localhost:9528) ──代理──> Spring Boot(localhost:8080)
   │                                                            │
   └──WebSocket──> ws://localhost:8080/ws/chat ──> OpenAI API
```

### 6.2 HTTP请求机制

**核心文件：** `vue-web/src/utils/request.js`

1. **Axios实例创建**：
   - `baseURL` 读取自环境变量 `VUE_APP_BASE_API`
   - 开发环境：`http://localhost:8080`（来自 `.env.development`）
   - 生产环境：`/prod-api`（来自 `.env.production`，需Nginx反向代理）
   - 请求超时：5000ms

2. **请求拦截器**：
   - 从Vuex Store获取Token
   - 在请求头添加 `X-Token` 字段携带JWT Token

3. **响应拦截器**：
   - 判断响应码 `code === 20000` 为成功
   - 其他码视为错误，弹出Element UI的Message提示
   - 特殊错误码（50008/50012/50014）表示Token失效，提示重新登录

4. **Token存储**：
   - 使用 `js-cookie` 库将Token存储在Cookie中
   - Cookie key: `vue_admin_template_token`

### 6.3 WebSocket连接机制（AI助手）

**核心文件：** `vue-web/src/views/ai/index.vue` + `x-admin/.../websocket/`

1. **连接建立**：
   - 前端在 `mounted()` 生命周期中创建WebSocket连接
   - 连接地址：`ws://localhost:8080/ws/chat`（**硬编码**）
   - 后端 `WebSocketConfig` 注册处理器，允许所有源（`setAllowedOrigins("*")`）

2. **消息发送**：
   - 用户输入消息后，构建JSON格式：`{type: "message", text: "...", username: "..."}`
   - 通过 `socket.send(JSON.stringify(message))` 发送

3. **消息接收（流式）**：
   - 后端通过Spring AI调用OpenAI API，以Flux流式返回数据
   - 每个数据块以 `data:` 前缀 + `\n\n` 后缀发送
   - 结束标记：`data:[DONE]\n\n`
   - 前端逐块追加到AI消息中，实现打字机效果

4. **重连机制**：
   - 最大重连次数：5次
   - 重连间隔：3000ms
   - 连接超时：5000ms

5. **特殊功能**：
   - 输入"AI健康建议"时，后端自动查询用户历史健康数据
   - 构建包含完整健康信息的Prompt发送给OpenAI
   - AI根据健康数据给出个性化建议

### 6.4 跨域处理

**后端CORS配置**（`CORSConfig.java`）：
- 允许所有路径跨域：`addMapping("/**")`
- 允许所有域名：`allowedOriginPatterns("*")`
- 允许Cookie：`allowCredentials(true)`
- 允许方法：GET、POST、DELETE、PUT
- 允许所有Header：`allowedHeaders("*")`
- 缓存时间：3600秒

## 七、项目访问方式与部署

### 7.1 当前开发环境访问方式

**仅限本机访问：**
- 前端：`http://localhost:9528`
- 后端API：`http://localhost:8080`
- WebSocket：`ws://localhost:8080/ws/chat`
- API文档：`http://localhost:8080/swagger-ui.html`

### 7.2 局域网内让他人访问

要让同一局域网内的其他人访问，需要以下步骤：

#### 步骤1：修改前端API地址

修改 `vue-web/.env.development`：
```
VUE_APP_BASE_API = 'http://你的局域网IP:8080'
```

#### 步骤2：修改WebSocket地址

修改 `vue-web/src/views/ai/index.vue` 中的WebSocket URL：
```javascript
const wsUrl = `ws://你的局域网IP:8080/ws/chat`
```

#### 步骤3：配置webpack-dev-server允许外部访问

修改 `vue-web/vue.config.js` 中的 devServer 配置：
```javascript
devServer: {
  port: port,
  open: true,
  host: '0.0.0.0',  // 添加此行，允许外部访问
  ...
}
```

#### 步骤4：开放防火墙端口

- 开放 **9528** 端口（前端开发服务器）
- 开放 **8080** 端口（后端API服务器）

#### 步骤5：获取本机局域网IP

- Windows: `ipconfig` 查看 IPv4 地址（如 `192.168.1.100`）
- 他人通过 `http://192.168.1.100:9528` 访问

### 7.3 生产环境部署方案

#### 方案一：Nginx反向代理（推荐）

```
用户浏览器 ──> Nginx(80/443) ──> 前端静态文件(dist/)
                            ──> 后端API(localhost:8080)
```

**Nginx配置示例：**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/vue-web/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /prod-api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket代理
    location /ws/ {
        proxy_pass http://localhost:8080/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

**前端构建命令：**
```bash
cd vue-web
npm run build:prod
```

构建产物在 `vue-web/dist/` 目录。

#### 方案二：Spring Boot内嵌前端

将前端构建产物放入后端 `src/main/resources/static/` 目录，通过Spring Boot直接提供静态文件服务。

## 八、系统功能模块详解

### 8.1 用户管理模块

- **注册**：POST `/user/register` → 默认分配普通用户角色
- **获取用户信息**：GET `/user/info?token=xxx`
- **用户列表**：GET `/user/list`（分页、支持按用户名/手机号搜索）

- **修改用户**：PUT `/user/update`
- **删除用户**：DELETE `/user/{id}`（逻辑删除）
- **修改密码**：PUT `/user/changePassword`

### 8.2 角色管理模块
- **角色列表**：GET `/role/list`（分页）
- **新增角色**：POST `/role`
- **修改角色**：PUT `/role`
- **删除角色**：DELETE `/role/{id}`
- **所有角色**：GET `/role/all`

### 8.3 菜单管理模块
- **所有菜单**：GET `/menu`（树形结构）

### 8.4 身体信息管理模块
- **上传身体信息**：POST `/user/BodyInformation`
- **保存历史记录**：POST `/user/BodyInformationNotes`
- **获取身体信息**：GET `/user/getBodyInfo`
- **身体信息列表**：GET `/user/getBodyList`（分页）
- **获取历史记录**：GET `/user/getBodyNotes/{id}`
- **用户身体历史列表**：GET `/user/getUserBodyList`（分页）

### 8.5 运动知识模块
- 运动类型与疾病关联详情（detail表）
- 运动建议信息（sport_info表）

### 8.6 AI健康助手模块
- WebSocket连接：`ws://localhost:8080/ws/chat`
- 普通对话：直接发送问题给OpenAI
- 健康建议：输入"AI健康建议"，系统自动查询用户历史健康数据构建Prompt

## 九、安全风险与注意事项

### 9.1 已发现的安全问题

| 问题 | 严重程度 | 位置 | 说明 |
|------|----------|------|------|
| 密码明文存储 | 🔴 高 | j_user表 | 密码未加密，直接存储明文 |
| JWT密钥硬编码 | 🔴 高 | JwtConfig.java | 密钥写在代码中，应使用配置文件 |
| OpenAI API Key暴露 | 🔴 高 | application.yml | base-url和api-key配置反了，且暴露在代码中 |
| WebSocket URL硬编码 | 🟡 中 | ai/index.vue | WebSocket地址写死为localhost |
| CORS全开放 | 🟡 中 | CORSConfig.java | 允许所有域名跨域 |
| 无接口级权限校验 | 🟡 中 | 所有Controller | 仅靠前端菜单控制，后端API无角色校验 |
| Redis未启用认证 | 🟡 中 | application.yml | Redis无密码保护 |
| 登录用户存实例变量 | 🟡 中 | UserServiceImpl | loginUser为实例变量，多线程下会串用户 |

### 9.2 application.yml配置错误

当前配置中OpenAI的base-url和api-key写反了：
```yaml
spring:
  ai:
    openai:
      base-url: sk-proj-zybt0sY...  # 这实际是API Key
      api-key: https://api.openai-proxy.com  # 这实际是Base URL
```

应修正为：
```yaml
spring:
  ai:
    openai:
      base-url: https://api.openai-proxy.com
      api-key: sk-proj-zybt0sY...
```

## 十一、总结

本项目是一个功能完整的AI健康管理系统，采用前后端分离架构，核心特点如下：

1. **权限体系**：基于RBAC模型，通过 `用户-角色-菜单` 三级关联实现权限控制，管理员和普通用户通过 `j_user_role` 中间表中的 `role_id` 值区分（1=admin, 3=normal）

2. **网络通信**：前端通过Axios发起HTTP请求（携带JWT Token），通过WebSocket实现AI对话的流式通信

3. **访问方式**：当前仅支持localhost访问，局域网访问需修改API地址、WebSocket地址和devServer配置；生产部署推荐Nginx反向代理方案

4. **AI集成**：通过Spring AI接入OpenAI，支持普通对话和基于用户健康数据的个性化健康建议
