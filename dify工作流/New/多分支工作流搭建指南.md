# AI健康助手 — Dify 多分支工作流搭建指南

---

## 一、架构概览

### 改造目标

将原有的3节点线性工作流升级为多分支架构，按用户意图自动路由到专用处理流程，提升回答精准度。

### 架构对比

```
改造前（线性）：
[开始] → [知识检索] → [LLM] → [回复]

改造后（多分支）：
[开始] → [意图分类] ─┬─ A: 运动咨询 → [知识检索] → [运动建议生成] ─┐
                      ├─ B: 饮食咨询 → [饮食检索] → [饮食建议生成] ─┤
                      ├─ C: 健康评估 → [数据提取] → [评估生成] ─────┤
                      ├─ D: 就医建议 → [拒绝话术] ─────────────────┤
                      └─ E: 闲聊/其他 → [通用回复] ─────────────────┤
                                                                    ├→ [变量聚合] → [格式化输出] → [结束]
```

### 工作流节点清单

| 序号 | 节点名称 | 节点类型 | 模型 | 说明 |
|------|----------|----------|------|------|
| 1 | 开始 | Start | - | 接收用户输入和健康数据 |
| 2 | 意图分类 | LLM | deepseek-v4-flash | 识别用户意图，输出A/B/C/D/E |
| 3 | 条件分支 | IF/ELSE | - | 根据意图分类结果路由 |
| 4 | 运动知识检索 | Knowledge Retrieval | - | 检索运动健康知识库 |
| 5 | 运动建议生成 | LLM | deepseek-v4 | 生成运动建议 |
| 6 | 饮食知识检索 | Knowledge Retrieval | - | 检索饮食健康知识库 |
| 7 | 饮食建议生成 | LLM | deepseek-v4 | 生成饮食建议 |
| 8 | 健康数据提取 | Code | - | 提取用户健康数据关键指标 |
| 9 | 评估生成 | LLM | deepseek-v4 | 生成健康评估报告 |
| 10 | 就医拒绝话术 | LLM | deepseek-v4-flash | 固定拒绝话术 |
| 11 | 通用回复 | LLM | deepseek-v4-flash | 闲聊引导 |
| 12 | 变量聚合 | Variable Aggregator | - | 汇总各分支输出 |
| 13 | 格式化输出 | Code | - | 统一输出格式 |
| 14 | 直接回复 | Answer | - | 返回结果 |


## 三、搭建步骤

### 步骤1：创建知识库

#### 知识库1：运动健康知识库

1. 进入 **知识库** → **创建知识库**
2. 名称：`运动健康知识库`
3. 上传文件：`运动健康知识库.txt`
4. 分段设置：

| 参数 | 值 |
|------|-----|
| 分段标识符 | `##` |
| 分段最大长度 | 500 tokens |
| 分段重叠 | 50 tokens |

5. 索引方式：**高质量模式**（使用Embedding模型）
6. 检索设置：

| 参数 | 值 |
|------|-----|
| 检索模式 | 混合检索（语义+关键词） |
| Top K | 3 |
| 分数阈值 | 0.5 |

7. 点击 **保存并处理**，等待索引完成

#### 知识库2：饮食健康知识库

1. 同上流程，名称：`饮食健康知识库`
2. 上传文件：`饮食健康知识库.txt`
3. 分段和检索设置同上

---

### 步骤2：创建工作流

1. 进入 **工作室** → **创建应用**
2. 类型选择：**Chatflow**
3. 名称：`AI健康助手-多分支版`
4. 描述：基于意图分类的多分支健康助手

---

### 步骤3：配置开始节点

点击 **开始** 节点，添加以下输入变量：

| 变量名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `user_message` | String | 是 | 用户输入的问题 |
| `username` | String | 是 | 用户名 |
| `health_data` | Object | 否 | 用户健康数据JSON |

---

### 步骤4：添加意图分类节点

1. 点击 **+** 添加节点 → 选择 **LLM**
2. 节点名称：`意图分类`
3. 模型选择：`deepseek-v4-flash`（快速低成本）
4. SYSTEM Prompt：

```
你是一个意图分类器。根据用户输入，判断其意图类别，只返回一个大写字母。

类别定义：
A - 运动咨询：询问某项运动是否适合、运动方法、运动建议、运动注意事项
B - 饮食咨询：询问饮食建议、营养搭配、食物禁忌、食疗方案
C - 健康评估：希望分析自己的健康数据、给出综合健康建议、身体状况评估
D - 就医建议：涉及疾病诊断、用药指导、治疗方案、症状分析
E - 闲聊/其他：打招呼、问系统功能、闲聊、无法归类的问题

判断规则：
- 如果问题同时涉及运动和饮食，选择更侧重的那个
- 如果问题涉及疾病诊断或用药，必须选择D
- 如果无法确定类别，选择E

只输出一个字母，不要输出任何其他内容。
```

5. USER Prompt：`{{#Start.user_message#}}`

---

### 步骤5：添加条件分支节点

1. 点击 **+** 添加节点 → 选择 **IF/ELSE**
2. 节点名称：`意图路由`
3. 配置条件：

**分支1：运动咨询**
```
IF 变量 {{#意图分类.text#}} 包含 A
```

**分支2：饮食咨询**
```
ELSE IF 变量 {{#意图分类.text#}} 包含 B
```

**分支3：健康评估**
```
ELSE IF 变量 {{#意图分类.text#}} 包含 C
```

**分支4：就医建议**
```
ELSE IF 变量 {{#意图分类.text#}} 包含 D
```

**分支5：闲聊/其他**
```
ELSE（以上都不满足）
```

### 步骤8：健康评估分支

#### 8.1 添加代码节点（提取健康数据）

1. 在 **IF C** 分支后添加 **代码执行** 节点
2. 节点名称：`健康数据提取`
3. 语言：Python3
4. 代码：

```python
import json

def main(health_data: dict) -> dict:
    if not health_data:
        return {"has_data": False, "summary": "用户未上传健康数据"}
    
    # 提取关键指标
    indicators = []
    
    # 血压
    bp = health_data.get("bloodPressure", "")
    if bp:
        indicators.append(f"血压: {bp}")
    
    # 血糖
    bs = health_data.get("bloodSugar", "")
    if bs:
        indicators.append(f"血糖: {bs}")
    
    # 血脂
    bl = health_data.get("bloodLipid", "")
    if bl:
        indicators.append(f"血脂: {bl}")
    
    # 心率
    hr = health_data.get("heartRate", "")
    if hr:
        indicators.append(f"心率: {hr}")
    
    # BMI
    height = health_data.get("height", 0)
    weight = health_data.get("weight", 0)
    if height and weight and height > 0:
        bmi = round(weight / ((height/100) ** 2), 1)
        indicators.append(f"BMI: {bmi}")
    
    # 睡眠
    sleep = health_data.get("sleepDuration", "")
    quality = health_data.get("sleepQuality", "")
    if sleep:
        indicators.append(f"睡眠时长: {sleep}, 质量: {quality}")
    
    # 生活习惯
    smoking = health_data.get("smoking", False)
    drinking = health_data.get("drinking", False)
    exercise = health_data.get("exercise", False)
    habits = []
    if smoking: habits.append("吸烟")
    if drinking: habits.append("饮酒")
    if exercise: habits.append("规律运动")
    if habits:
        indicators.append(f"生活习惯: {', '.join(habits)}")
    
    summary = "；".join(indicators) if indicators else "无有效健康数据"
    
    return {
        "has_data": bool(indicators),
        "summary": summary,
        "age": health_data.get("age", "未知"),
        "gender": health_data.get("gender", "未知")
    }
```

5. 输入变量：`health_data` = `{{#Start.health_data#}}`

```
{{#健康数据提取.summary#}}
```

### 步骤12：添加格式化输出节点

1. 在变量聚合后添加 **代码执行** 节点
2. 节点名称：`格式化输出`
3. 语言：Python3
4. 代码：

```python
def main(运动回复: str, 饮食回复: str, 评估回复: str, 就医回复: str, 闲联回复: str) -> dict:
    # 取非空的回复
    for reply in [运动回复, 饮食回复, 评估回复, 就医回复, 闲联回复]:
        if reply and reply.strip():
            return {"result": reply.strip()}
    
    return {"result": "抱歉，我暂时无法回答这个问题，请换个问法试试。"}
```

### 步骤13：添加结束节点

1. 添加 **直接回复** 节点
2. 回复内容：`{{#格式化输出.result#}}`

### 步骤14：连接节点

按以下顺序连接所有节点：

```
开始 → 意图分类 → 意图路由
意图路由 IF A → 运动知识检索 → 运动建议生成 → 汇总输出
意图路由 IF B → 饮食知识检索 → 饮食建议生成 → 汇总输出
意图路由 IF C → 健康数据提取 → 评估生成 → 汇总输出
意图路由 IF D → 就医拒绝话术 → 汇总输出
意图路由 ELSE → 通用回复 → 汇总输出
汇总输出 → 格式化输出 → 直接回复
```

## 五、优化建议

### 5.1 意图分类优化

如果分类准确率不高，可以：
1. 增加few-shot示例到SYSTEM Prompt
2. 调整分类边界描述
3. 使用更强的模型（如deepseek-v4）做意图分类

### 5.2 知识库优化

1. 定期更新知识库内容
2. 根据用户高频问题补充知识条目
3. 调整Top K和分数阈值优化检索效果

### 5.3 Prompt优化

1. 根据实际对话效果调整SYSTEM Prompt
2. 增加更多回答示例
3. 针对边界case添加特殊处理规则
